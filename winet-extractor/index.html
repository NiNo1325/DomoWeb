Configurer deux onduleurs Sungrow dans Home Assistant avec WiNet Extractor

Faire tourner deux instances de l'add-on WiNet Extractor via MQTT pour superviser deux onduleurs Sungrow — guide pas à pas.

Retour d'expérience. Deux onduleurs Sungrow supervisés dans Home Assistant en faisant tourner deux instances de l'add-on WiNet Extractor, chacune pointant vers un WiNet S / S2 distinct et publiant ses mesures via MQTT. Résultat : production instantanée, journalière et totale des deux onduleurs, intégrées au tableau de bord Énergie.

1. Introduction à WiNet Extractor

WiNet Extractor est un add-on Home Assistant conçu pour récupérer les données des onduleurs Sungrow WiNet S / WiNet S2.

Il permet de :

  • lire la puissance instantanée de l'onduleur ;
  • suivre la production journalière et totale ;
  • afficher des valeurs utiles : tension, courant, température interne.

Dans ce scénario, l'objectif est d'installer deux instances de WiNet Extractor afin de superviser deux onduleurs distincts.

2. Prérequis et préparation

Avant de commencer, assurez-vous de disposer :

  • d'une installation Home Assistant fonctionnelle (dernière version recommandée) ;
  • de l'accès à l'interface Supervisor (pour installer des add-ons) ;
  • de l'adresse IP et des identifiants de vos onduleurs Sungrow ;
  • d'un ordinateur connecté au même réseau local que Home Assistant ;
  • de l'accès à la boutique d'add-ons et aux Paramètres de Home Assistant.

3. Télécharger WiNet Extractor depuis GitHub

  1. Rendez-vous sur le dépôt GitHub officiel NickStallman/home-assistant-repo (merci au développeur pour sa contribution).
  2. Cliquez sur Code → Download ZIP.
  3. Décompressez le fichier ZIP sur votre ordinateur.
  4. À l'intérieur, repérez le dossier winet-extractor — c'est ce dossier qu'il faut transférer vers Home Assistant.

Conseil. Conservez une copie du ZIP au cas où une réinstallation serait nécessaire plus tard.

4. Installer Samba Share pour transférer un dossier complet

L'éditeur de fichiers intégré à Home Assistant ne permet pas de téléverser des dossiers directement. Samba Share permet d'accéder aux fichiers de Home Assistant depuis l'explorateur de fichiers de votre PC.

  1. Dans Home Assistant, ouvrez la boutique d'add-ons, cherchez Samba Share et installez-le.
  2. Configurez un nom d'utilisateur et un mot de passe, puis démarrez l'add-on.
  3. Sous Windows, ouvrez l'Explorateur de fichiers et saisissez \\IP_DE_HOME_ASSISTANT\config (avec vos identifiants).
  4. Ouvrez le dossier addons (créez-le s'il n'existe pas).
  5. Glissez-déposez le dossier winet-extractor dans /addons/.

Conseil. Si l'add-on File Editor n'affiche pas /addons, ajoutez ce chemin dans allowlist_external_dirs puis redémarrez l'éditeur.

5. Ajouter WiNet Extractor comme dépôt local dans Home Assistant

  • Dans /addons sur Home Assistant, créez un nouveau dossier si nécessaire.
  • Renommez le dossier winet-extractor en winet-extractor2.
  • Glissez-déposez le dossier winet-extractor2 dans /addons.
  • Ouvrez le fichier config.yaml dans winet-extractor2 et modifiez :
    • name: → Winet Extractor 2 ;
    • slug: → winet-extractor2.
  • Dans Home Assistant, allez dans Paramètres → Add-ons → Dépôts et actualisez (via le menu en haut à droite).

Le changement de name et de slug est important : il indique à Home Assistant qu'il s'agit de deux modules distincts.

6. Installer et configurer la première instance

  1. Installez le WiNet Extractor original (depuis le dépôt standard — pas celui créé localement).
  2. Réglez les options :
    • WINET_HOST = IP du WiNet de l'onduleur n°1 (à trouver dans votre routeur) ;
    • MQTT_URL = mqtt://localhost (ou l'adresse de votre broker) ;
    • WINET_USER / WINET_PASS = identifiants admin du WiNet.
  3. Démarrez l'add-on.

7. Installer et configurer la seconde instance

  1. Ouvrez « WiNet Extractor 2 » dans la boutique d'add-ons (l'add-on local, en haut de l'écran).
  2. Installez-le, puis configurez-le avec les valeurs du second onduleur :
    • WINET_HOST = IP du WiNet de l'onduleur n°2 ;
    • mêmes MQTT_URL et identifiants que ci-dessus.
  3. Démarrez l'add-on.

Optionnel. Pour bien séparer les topics, définissez des préfixes de topic ou des client IDs différents dans chaque instance (si votre version le permet). Pour la journalisation, l'auteur a créé deux utilisateurs MQTT (un par instance), mais c'est facultatif.

8. Ajouter les capteurs au tableau de bord Énergie

  1. Ouvrez votre tableau de bord Énergie (menu en haut à droite pour configurer les capteurs).
  2. Sous Panneaux solaires, ajoutez l'entité Production totale (Total Yield) de l'onduleur n°1.
  3. Répétez l'opération pour l'onduleur n°2.

Pourquoi la production totale et pas la journalière ? Le tableau de bord Énergie attend un compteur cumulatif (state_class: total_increasing). La production journalière se remet à zéro chaque jour et ne convient pas aux graphiques de long terme.

La section Solaire apparaît alors dans le tableau de bord Énergie avec les deux onduleurs. Si votre installation dispose d'une vue de consommation globale (par exemple via un Shelly 3EM Pro), elle calcule automatiquement l'autoconsommation et les indicateurs d'autosuffisance.

9. Dépannage

  • L'add-on local n'apparaît pas : vérifiez que config.yaml possède un slug unique et que le dossier se trouve directement sous /addons/.
  • Erreur d'installation : assurez-vous que tous les fichiers (Dockerfile, scripts) du dossier winet-extractor original ont bien été copiés.
  • Aucune donnée : vérifiez l'IP et les identifiants du WiNet, et confirmez que votre broker MQTT fonctionne.
