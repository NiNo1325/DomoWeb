Retour d’expérience reproductible. Ce guide décrit une intégration réellement montée et validée : un régulateur de piscine Syclope HydroTouch CHY0444 est lu en Modbus RTU via son port RS485, encapsulé dans TCP par un serveur série Waveshare RS232/485/422 TO POE ETH (B), puis intégré nativement dans Home Assistant.
Les deux mesures finales ont été vérifiées simultanément sur l’écran du Syclope et par lecture logicielle : pH ≈ 7,17 et Redox ≈ 882 mV.

. Les adresses IP visibles sont des exemples, les adresses MAC et identifiants matériels ont été masqués, et aucune donnée d’authentification n’est publiée. Matériel testé : Syclope CHY0444 et Waveshare firmware V1.452 ; validation réalisée le 11 juillet 2026.

2. Sécurité et limites

Cette installation mélange :

  • du 230 V AC dans un coffret ;
  • une alimentation TBTS 12 V DC ;
  • des produits chimiques de piscine ;
  • un régulateur pilotant des pompes de dosage.

Règles impératives

  1. Couper et consigner l’alimentation avant d’ouvrir le coffret.
  2. Vérifier l’absence de tension avec un appareil adapté.
  3. Faire réaliser le câblage secteur par une personne qualifiée.
  4. Utiliser des embouts de câblage, des presse-étoupes et une séparation correcte entre secteur et basse tension.
  5. Ne pas se fier uniquement à la couleur des conducteurs. Dans une installation belge 3 × 230 V sans neutre, un conducteur bleu peut être actif selon le câblage.
  6. Ne jamais modifier un registre Modbus inconnu. Le scan et l’intégration présentés ici sont en lecture seule.
  7. Ne pas utiliser Home Assistant comme unique dispositif de sécurité du traitement. Les sécurités hydrauliques, de débit et de surdosage doivent rester gérées par le Syclope et l’installation de filtration.

La notice Syclope impose d’ailleurs de couper l’alimentation primaire avant le raccordement RS485 et indique que le port est destiné à l’enregistrement des mesures, alarmes et états.1

4. Schéma de câblage

4.1 Alimentation 230 V → 12 V DC

L’alimentation DIN possède :

  • en entrée : L et N ;
  • en sortie : +V et -V ;
  • une plage d’ajustement d’environ 10,8 à 13,8 V sur le modèle observé.

Dans le montage réalisé, l’entrée de l’alimentation a été reprise en aval d’un Schneider C10 existant.

Avant de raccorder le Waveshare :

  1. alimenter brièvement l’alimentation seule ;
  2. mesurer au multimètre entre +V et -V ;
  3. régler la sortie autour de 12,0 V ;
  4. couper à nouveau le secteur ;
  5. connecter +V → VCC et -V → GND.

Le bornier du Waveshare est clairement repéré :

RB | RA | TA | TB | PE | GND | VCC

Pour l’alimentation utilisée ici :

  • fil rouge : VCC ;
  • fil noir : GND.

4.2 Raccordement RS485 côté Syclope

Le bornier RS485 est situé en haut à gauche de la carte HydroTouch et repéré verticalement :

  • C
  • B
  • A

La notice officielle demande de câbler A, B et C.4 Dans la liaison courte testée, A et B seuls ont suffi. Pour une installation plus robuste, surtout avec un câble long ou un environnement perturbé :

  • Syclope A → Waveshare TA ;
  • Syclope B → Waveshare TB ;
  • Syclope C → Waveshare GND comme conducteur de référence, après vérification avec le fabricant.

Le mapping A/B n’est pas universel entre fabricants. Si aucune réponse n’est obtenue alors que tout le reste est correct, couper l’alimentation et inverser A/B.

4.3 Choix du câble

Un câble réseau U/UTP à quatre paires AWG24 a été utilisé.

Utiliser les deux conducteurs d’une même paire torsadée :

  • un conducteur pour A ;
  • l’autre pour B.

Ne pas prendre A dans une paire et B dans une autre. Garder les torsades au plus près des borniers. Les autres conducteurs doivent être isolés et rangés.

Pour une liaison longue :

  • préférer un câble blindé pour bus industriel ;
  • mettre une terminaison 120 Ω aux deux extrémités uniquement lorsque la topologie et la longueur le justifient ;
  • éviter les dérivations en étoile.

Waveshare précise que, sur une courte distance, ses résistances de terminaison peuvent être laissées hors circuit, tandis que les longues liaisons multi-équipements utilisent typiquement 120 Ω aux deux extrémités.5

4.4 Ethernet

Le câble RJ45 du Waveshare rejoint le réseau local, directement ou via un adaptateur CPL.

Une fois sous tension :

  • LED Power rouge fixe ;
  • LED Link verte lorsque le lien Ethernet est établi ;
  • LED Active lors des échanges série.

7. Configuration du Waveshare

7.1 Découverte avec VirCom

VirCom recherche les serveurs série par diffusion réseau. Il peut donc voir un appareil qui n’est pas encore dans le même sous-réseau IP que le PC, alors que arp -a peut ne rien montrer.

Pourquoi arp -a n’était pas concluant

La table ARP :

  • n’est pas un scanner réseau ;
  • ne contient que des voisins récemment résolus ;
  • ne montre pas forcément un appareil sur un autre sous-réseau ;
  • peut rester identique lorsqu’un appareil est débranché tant que l’entrée n’expire pas.

VirCom utilise sa propre découverte par broadcast, ce qui explique qu’il ait trouvé le Waveshare alors que la table ARP ne le révélait pas.

7.3 Paramètres série

Paramètre Valeur
Baud Rate 19200
Data bits 8
Parity None
Stop bits 1
Flow control None

Le point essentiel découvert pendant le diagnostic est le framing réseau :

  • ce n’est pas du Modbus TCP natif ;
  • ce sont des trames Modbus RTU brutes encapsulées dans une connexion TCP ;
  • le client doit donc utiliser RTU over TCP.

Le Waveshare est utilisé comme serveur série TCP transparent. Le constructeur documente le mode TCP Server, le port de données 4196 dans ses exemples et la prise en charge de la transmission série/Ethernet.6

8. Deuxième boucle de vérification : réseau

Dans PowerShell :

Test-NetConnection 192.168.1.200 -Port 4196
Test-NetConnection 192.168.1.200 -Port 502

Le résultat attendu pour chaque port est :

TcpTestSucceeded : True

Un TcpTestSucceeded : True prouve uniquement que le socket TCP est ouvert. Il ne prouve pas encore que le Syclope répond en Modbus.

En cas d’appareil introuvable

  1. connecter temporairement le PC directement au Waveshare ;
  2. désactiver le Wi-Fi du PC ;
  3. attribuer au PC une IP statique compatible avec l’IP d’usine, par exemple 192.168.1.50/24 ;
  4. lancer VirCom ;
  5. attribuer au Waveshare une IP du réseau principal ;
  6. remettre le PC en DHCP.

9. Validation avec Modbus Poll

9.1 Connexion

Dans Modbus Poll, choisir :

  • connexion : Modbus RTU/ASCII Over TCP/IP ;
  • mode : RTU ;
  • IP : 192.168.1.200 ;
  • port : 4196.

Ne pas choisir le Modbus TCP classique pour cette configuration.

9.2 Première lecture

Dans Setup → Read/Write Definition :

Champ Valeur

Une réponse avec des valeurs numériques, même incompréhensibles, confirme :

  • le TCP ;
  • le framing RTU ;
  • le câblage RS485 ;
  • l’adresse esclave ;
  • la vitesse et la parité.

9.3 Test de la fonction 04

La fonction 04 Read Input Registers a renvoyé :

01 Illegal Function

C’est une information utile, pas une panne :

  • la requête est bien arrivée au Syclope ;
  • le Syclope a répondu ;
  • cette fonction n’est simplement pas implémentée pour ce besoin.

La suite des recherches s’est donc faite exclusivement avec la fonction 03.

10. Rétro-ingénierie de la cartographie Modbus

La table de registres détaillée du CHY0444 n’étant pas disponible publiquement, les registres ont été identifiés par lecture systématique et corrélation avec l’écran.

10.1 Premier échec Python : Modbus TCP classique

Le client TCP se connectait :

Connexion TCP : True

mais ne recevait aucune réponse à la requête. La cause était le framer par défaut de ModbusTcpClient, qui envoie des trames Modbus TCP avec en-tête MBAP.

Le Waveshare attendait ici des trames RTU brutes.

10.2 Correction : framer RTU

La correction décisive :

from pymodbus import FramerType
from pymodbus.client import ModbusTcpClient

client = ModbusTcpClient(
    host="192.168.1.200",
    port=4196,
    framer=FramerType.RTU,
    timeout=3,
    retries=0,
)

La même lecture a alors renvoyé exactement les valeurs observées dans Modbus Poll.

10.3 Scan des registres

Le script fourni dans scripts/scan_syclope.py lit les Holding Registers par blocs et génère :

  • un CSV brut ;
  • un CSV de floats plausibles ;
  • un CSV des zones répondant.

Commande avec l’adresse d’exemple :

pip install -r scripts\requirements.txt
python scripts\scan_syclope.py --host 192.168.1.200 --start 0 --end 5000 --block 100

Les scripts acceptent également SYCLOPE_HOST, SYCLOPE_PORT et SYCLOPE_DEVICE_ID comme variables d’environnement. Un modèle est fourni dans .env.example.

Le scan réel a identifié des zones actives notamment :

  • 0–399 ;
  • 1000–1499 ;
  • 2000–3299.

Les fichiers bruts sont fournis dans le dossier data/.

10.4 Comprendre l’adressage

Modbus Poll affichait :

Address 1202  → PLC address 41203

La conversion utilisée est :

adresse PLC = 40001 + adresse 0-based

Donc :

Donnée Registre PLC Adresse Home Assistant / PyModbus
pH 41203 1202
Redox 41303 1302

10.5 Comprendre le float32 CDAB

Chaque mesure occupe deux registres 16 bits.

Pour le pH, un scan a donné :

  • adresse 1202 : 0x8D55 ;
  • adresse 1203 : 0x40E5.

L’ordre brut n’est pas ABCD mais CDAB : les deux mots 16 bits sont inversés.

En Python :

raw = second.to_bytes(2, "big") + first.to_bytes(2, "big")
value = struct.unpack(">f", raw)[0]

Dans Home Assistant, l’équivalent est :

data_type: float32
swap: word

La documentation Home Assistant définit précisément swap: word comme la transformation ABCD → CDAB et indique qu’un float32 occupe deux registres.7

13. Intégration dans Home Assistant

Home Assistant prend en charge explicitement le type rtuovertcp, destiné aux forwarders TCP vers liaison série.8

Ajouter dans configuration.yaml :

modbus:
  - name: syclope_hydrotouch
    type: rtuovertcp
    host: 192.168.1.200
    port: 4196
    timeout: 5
    message_wait_milliseconds: 100

    sensors:
      - name: "Syclope pH"
        unique_id: syclope_ph
        slave: 1
        address: 1202
        input_type: holding
        data_type: float32
        swap: word
        precision: 2
        scan_interval: 15
        state_class: measurement
        unit_of_measurement: "pH"

      - name: "Syclope Redox"
        unique_id: syclope_redox
        slave: 1
        address: 1302
        input_type: holding
        data_type: float32
        swap: word
        precision: 0
        scan_interval: 15
        state_class: measurement
        unit_of_measurement: "mV"

Le même bloc est disponible dans :

home-assistant/configuration_modbus.yaml

Pourquoi il n’y a pas icon:

L’option icon n’est pas acceptée dans la définition d’un capteur Modbus. L’icône peut être :

  • définie depuis l’interface de l’entité ;
  • ou imposée dans la carte du dashboard.

Après modification :

  1. Outils de développement → YAML → Vérifier la configuration ;
  2. redémarrer Home Assistant, ou utiliser le rechargement Modbus si disponible ;
  3. vérifier les entités.

Les identifiants confirmés sont :

  • sensor.syclope_ph
  • sensor.syclope_redox

17. Fiabilisation pour une installation durable

Réseau

  • réserver l’IP 192.168.1.200 dans le routeur, même si elle est statique ;
  • sauvegarder les paramètres VirCom/Waveshare ;
  • documenter le port 4196 ;
  • limiter l’accès au réseau local ;
  • ne pas exposer directement le Waveshare à Internet.

Électricité

  • utiliser un coffret fermé ;
  • ajouter un repérage clair des conducteurs ;
  • fixer le Waveshare sur rail ou support ;
  • éviter que les câbles exercent une traction sur les borniers ;
  • protéger la liaison Ethernet contre l’humidité ;
  • envisager une protection surtension dans un environnement extérieur.

Modbus

  • conserver un intervalle de lecture raisonnable, ici 15 s ;
  • ne pas scanner 65 536 registres en continu ;
  • ne pas écrire dans des registres non documentés ;
  • conserver les CSV d’origine pour comparer les futures versions de firmware.

Entretien de la mesure

L’intégration ne remplace pas :

  • la calibration des sondes ;
  • les contrôles colorimétriques ;
  • la vérification du débit ;
  • le remplacement des sondes vieillissantes ;
  • la surveillance des bidons et pompes.

Le constructeur recommande de confronter régulièrement les mesures affichées à une analyse de référence.10
